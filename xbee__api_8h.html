void xbee_Init ( xbee_RcvModemStatusFunc  rcvModemStatusFunc,
xbee_RcvAtCommandResponseFunc  rcvAtCommandResponseFunc,
xbee_RcvTxStatusFunc  rcvTxStatusFunc,
xbee_RcvRfData64Func  rcvRfData64Func,
xbee_RcvRfData16Func  rcvRfData16Func,
xbee_RcvRfDataByteFunc  rcvRfDataByteFunc,
xbee_SendBytesFunc  sendBytesFunc,
xbee_FlushFunc  flushFunc 
)

This function inits the different pointers to call-back functions to call when packets are received and to functions to call to send data through communication.

Warning: all these functions are called during interruptions, so they have to be very short !

Parameters
rcvModemStatusFuncA pointer to the function to call when a modem status message is received. Can be null.
rcvAtCommandResponseFuncA pointer to the function to call when an AT command response is received. Can be null.
rcvTxStatusFuncA pointer to the function to call when a TX status message is received. Can be null.
rcvRfData64FuncA pointer to the function to call when an incoming RF message with 64-bits address is received. Can be null.
rcvRfData16FuncA pointer to the function to call when an incoming RF message with 16-bits address is received. Can be null.
rcvRfDataByteFuncA pointer to the function to call each time an incoming RF byte arrives. Warning: this function sends each byte of the message content directly, it does not check its validity (no checksum checked). Can be null.
sendBytesFuncA pointer to the function to call to send bytes through communication. Can NOT be null.
flushFuncA pointer to the function to call to flush data on the communication. Can be null.

void xbee_SetDestinationAndOptions ( unsigned short  destination,
unsigned char  options 
)

This function sets the packet destination and options.

This function sets the destination and the options of all the future packet to be flushed. The current packet is first flushed to the current destination (before modification)

Parameters
destinationThe 16-bits destination address of the packet, 0xFFFF means broadcast to all card in the same PAN Id.
optionsThe options for the packet (flags) : 0x01 = Disable ACK, 0x04 = Send packet with broadcast PAN Id. All other bits must be set to 0.
